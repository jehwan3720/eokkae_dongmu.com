## 3. UI/UX 설계

### 3-1. 그리드 레이아웃

```
모바일  : grid-cols-1 (세로 스택)
태블릿  : grid-cols-3 (sm breakpoint에서 바로 3열)
데스크탑: grid-cols-3 (lg는 그대로 3열)
```

> 기존 `sm:grid-cols-2`를 `sm:grid-cols-3`으로 변경.  
> 이유: 3개 카테고리에서 2열이면 마지막 항목이 하단 좌측에 단독 배치되어 레이아웃이 어색함.  
> `sm:grid-cols-3`으로 하면 태블릿에서도 3개가 나란히 정렬되어 깔끔함.

### 3-2. 카테고리 타이포그래피 (변경 없음)

```
카테고리 제목 : text-[0.6875rem] font-bold tracking-[0.14em] uppercase text-[var(--color-brand)]
항목 불릿     : w-1 h-1 rounded-full bg-neutral-300 (mt-[0.35rem])
항목 텍스트   : text-[0.8125rem] text-neutral-500 leading-snug
```

### 3-3. 확장 배경 (변경 없음)

```
bg-[#F9FAFB] border-t border-neutral-100 px-10 py-10
```

### 3-4. 애니메이션 (변경 없음)

```
AnimatePresence: height 0→auto / opacity 0→1
duration: 0.45s / ease: [0.16, 1, 0.3, 1]
ChevronDown rotate: 0→180 (0.3s)
```

## 5. 트레이드 오프 분석

| 이슈 | 현황 | 해결 방안 |
|------|------|-----------|
| 아코디언 확장 시 레이아웃 밀림 | 카드 내부 확장이므로 하단 섹션이 밀릴 수 있음 | `AnimatePresence + height:auto` 패턴은 자연스러운 밀림을 주어 UX상 문제 없음. `overflow-hidden`으로 확장 도중 가로 스크롤 없음 |
| 3열에서 항목 수 불균형 | 연구 3항목 / 공공 2항목 / 전시 3항목 | CSS Grid는 행 높이를 가장 긴 열 기준으로 자동 정렬 — 별도 처리 불필요 |
| 긴 텍스트 줄바꿈 | "화천군 생태 오염원 저감 습지(Biotope) 관리장 역임" | `leading-snug` + `items-start gap-2` 조합으로 자연스러운 멀티라인 허용 |
| 모바일 3열 좁음 | sm(640px) 뷰포트에서 3열 텍스트가 좁을 수 있음 | 각 항목이 짧은 편(최대 24자)이라 실용적으로 문제없음. 필요시 다시 `sm:grid-cols-2`로 되돌리는 것도 가능 |

## 7. UI 구조 상세 설계

### 7-1. '더 보기' 버튼 위치

```
┌──────────────────────────────────────────────────┐
│  메인 카드 (프로필 좌측 + About/Vision 우측)       │
├──────────────────────────────────────────────────┤  ← border-t border-neutral-100
│  [ 상세 경력 더 보기  ∨ ]  (w-full, 중앙 정렬)    │  ← 클릭 시 토글
├──────────────────────────────────────────────────┤
│  ┌─────────────┬─────────────┬─────────────┐    │
│  │ 연구 및 학술 │ 공공 및 관리 │ 전시 및 교육│    │
│  │ · 항목 1    │ · 항목 1    │ · 항목 1    │    │
│  │ · 항목 2    │ · 항목 2    │ · 항목 2    │    │
│  └─────────────┴─────────────┴─────────────┘    │
└──────────────────────────────────────────────────┘
```

- 버튼은 카드 본문과 확장 영역 사이의 경계선 역할을 겸함
- `border-t border-neutral-100`으로 시각적 구분
- 확장 상태에서 버튼 텍스트는 "상세 경력 접기"로 전환, ChevronDown 아이콘 180° 회전
- 호버 시: `hover:text-[var(--color-brand)] hover:-translate-y-0.5 transition-all duration-200`

### 7-2. 3단 카테고리 그리드

```
모바일(< 640px)  : 단일 열 세로 스택
태블릿+ (≥ 640px): sm:grid-cols-3 (즉시 3열)
데스크탑        : 동일 3열 유지
```

- `gap-10`으로 KMAC 특유의 넉넉한 열 간격
- 각 카테고리 블록은 독립적으로 높이 결정 (align-items: start)

## 8. 애니메이션 설계

### 8-1. 라이브러리

**Framer Motion** `AnimatePresence` + `motion.div` 사용.  
Next.js App Router 환경에서 CSS-only 아코디언(`max-height` 트릭)은 `height: auto` 계산 불가로 어색한 점프가 발생하므로 Framer Motion의 레이아웃 계산이 필수.

### 8-2. 확장 파라미터

| 속성 | 값 | 이유 |
|------|----|------|
| `height` | `0 → "auto"` | 콘텐츠 높이를 사전에 계산하지 않고 자동 측정 |
| `opacity` | `0 → 1` | 콘텐츠가 나타나는 느낌을 부드럽게 보완 |
| `duration` | `0.45s` | 300ms는 너무 빠르고 600ms는 답답함 — 중간값 |
| `ease` | `[0.16, 1, 0.3, 1]` | ease-out-expo 계열. 초반 빠르게 열리고 끝에서 감속 |
| `initial={false}` | AnimatePresence | 최초 마운트 시 진입 애니메이션 생략 |

### 8-3. ChevronDown 회전

```tsx
<motion.span animate={{ rotate: expanded ? 180 : 0 }} transition={{ duration: 0.3, ease: [0.16, 1, 0.3, 1] }}>
```

버튼 클릭과 동시에 회전 시작 → 확장 완료 전에 이미 180°에 도달하여 사용자에게 즉각적인 피드백 제공.

## 9. 타이포그래피 전략 (KMAC 스타일)

KMAC의 정갈함은 **여백과 위계의 명확한 분리**에서 온다. 텍스트가 길어져도 가독성을 유지하는 3단 계층 구조:

### 9-1. 계층 구조

```
레벨 1 — 카테고리 제목
  font-semibold | 0.6875rem | tracking-[0.14em] | uppercase
  color: var(--color-brand)  →  브랜드 색으로 섹션 앵커 역할
  margin-bottom: 1rem (mb-4)

레벨 2 — 불릿 마커
  w-1 h-1 rounded-full bg-neutral-300
  mt-[0.35rem] flex-shrink-0  →  텍스트 첫 줄 중앙 정렬 유지

레벨 3 — 항목 텍스트
  font: normal | 0.8125rem | text-neutral-500
  leading-relaxed  →  1.625 배율로 멀티라인 시 행간 충분히 확보
```

### 9-2. 장문 텍스트 처리

- `items-start gap-2` 조합으로 불릿이 첫 줄에 고정되고 텍스트만 줄바꿈
- `leading-relaxed`(1.625)는 `leading-snug`(1.375) 대비 +18% 행간 → 밀도 높은 내용도 호흡감 확보
- 최대 2~3항목 설계로 단일 카테고리가 과도하게 길어지지 않도록 데이터 수 제한

### 9-3. 배경색 분리

- 확장 영역: `bg-[#F9FAFB]` (메인 카드 `bg-white`보다 살짝 어두운 회색)
- 목적: 더 보기 영역이 "보조 정보"임을 시각적으로 구분하면서 섹션 배경(`bg-neutral-50`)과도 충돌하지 않음
